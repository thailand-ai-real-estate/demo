タイ不動産 LINE 業務 OS

LINE の問い合わせを、対応漏れなく案件に変える

タイの不動産取引は LINE が主戦場。届いた問い合わせを AI が顧客・条件・物件候補に整理し、対応漏れを防ぎ、返信案まで用意する業務 OS。普段どおり LINE を使うだけで、帰属・分配・証跡が裏の台帳に貯まる(co-broke は前面に出さない)。

顧客向け(需要入口)

物件 URL 確認

見つけた物件 URL を送ると、担当が掲載者・オーナーに確認して結果を返す入口。

物件診断レポート

AI 総合評価・価格妥当性・外国人購入適性・内見チェック(AI ドラフト＋担当確認)。

顧客マイページ

確認済み候補・反応(興味/内見/見送り)・内見予約。多言語。

業務 OS(社内・中核)

受信箱(LINE 業務 OS)

LINE 問い合わせを AI が条件・候補に整理。対応漏れ警告・返信案・台帳への確定。design partner の主役画面。

Deal OS(全体像)

問い合わせ〜成約を案件で一元管理。AI 要約・次アクション・候補・進行ステージ。

物件受託(最小版)

オーナーから物件を預かる起点。まずは最小フィールドだけ。

Verified Listing

同じ物件を内部ビュー(信頼度/流通)と顧客ビュー(募集中/外国人OK)で出し分け。

紹介登録(帰属)

誰が・どの顧客を・どの Listing へ。帰属を台帳に記録し報酬を保護。差別化の核。

Transaction Room

内見→交渉→契約→入居の進行と証跡。参加者・書類・AI 要約。

コミッション台帳

総額・分配・期限・証憑を記録。資金は預からず銀行振込。確実な支払いを可視化。

オーナー / エージェント向け

Owner Pipeline

供給オーナー向け。今どれだけ動いているか＋次に何を判断すべきか。多言語。

Agent Portal(将来)

客付けエージェント向け。客付け可能 Listing・確実な支払い・信頼スコア。

公開面(着地・ショーケース)

公開物件ショーケース

確認済みの少数物件だけを見せる公開ページ。ポータルではない。

公開物件ページ(共有用)

LINE/FB/広告に渡す着地ページ。募集中・内見可能・外国人OK・確認日。

法人オールインワン(Koki ビジョン / 将来構想・MVP対象外)

オールインワン(法人ダッシュボード)

Koki ビジョン全体像。保有物件・社宅・ドライバーまで1画面で管理する将来構想の入口。

財務・利回り

デポジット・家賃・付随サービスを台帳化し、利回りを自動算出。複数物件オーナー向け。

状態管理

室内備品・ダメージ・入退去点検を記録して共有。写真と状態を残す。

社宅管理(人事)

社員の契約・更新・解約・返金・満足度・クレームを管理。スコープ外の将来構想。

ドライバー管理

現在地・出勤・OT 自動記録・車両コンディション。不動産外のスコープ外構想。

デモ画面の入口です。各画面は設計ドキュメント(docs/)に基づくモックです。
